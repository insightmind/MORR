MORR KeyboardModule Testpage

This webpage contains sample cases for every event generated by the MORR KeyboardModule.
You can find a description of the expected behavior in each section.

Configuration:

  • EnabledModules must contain "MORR.Modules.Keyboard.KeyboardModule"
You can also use this preset configuration.

Test description

Each test consists of two items:
Action describes the action you (the tester) must perform in order to generate the relevant test data.
Expected behavior describes the event which should have resulted from your actions and thus be serialized into the recording file.
The following tests are grouped by event type.

In additional to the data described in the Expected behavior of each respective test, each event is also expected to hold this data:

  • TimeStamp: The serialized time of the event's occurence, e.g. "2020-03-16T17:43:52.517Z"
  • IssuingModule: "fab5bc0d-8b33-4dfd-9fa3-c58e0f1435b5"
If additional fields are stored but not mentioned, their data is not relevant to the test.

1. KeyboardInteractEvent

Test 1

Covers testcase(s): TC480

Action:
Enter this character sequence into the textbox below: akylo

Expected behavior:
The following KeyboardInteractEvents are created in the listed order:

Event 1:
  • PressedKeyName: "A"
  • ModifierKeys_System_Windows_Input_ModifierKeys: 0
  • ModifierKeysName: "None"
  • MappedCharacter_Unicode: "a"
Event 2:
  • PressedKeyName: "K"
  • ModifierKeys_System_Windows_Input_ModifierKeys: 0
  • ModifierKeysName: "None"
  • MappedCharacter_Unicode: "k"
Event 3:
  • PressedKeyName: "Y"
  • ModifierKeys_System_Windows_Input_ModifierKeys: 0
  • ModifierKeysName: "None"
  • MappedCharacter_Unicode: "y"
Event 4:
  • PressedKeyName: "L"
  • ModifierKeys_System_Windows_Input_ModifierKeys: 0
  • ModifierKeysName: "None"
  • MappedCharacter_Unicode: "l"
Event 5:
  • PressedKeyName: "O"
  • ModifierKeys_System_Windows_Input_ModifierKeys: 0
  • ModifierKeysName: "None"
  • MappedCharacter_Unicode: "o"
