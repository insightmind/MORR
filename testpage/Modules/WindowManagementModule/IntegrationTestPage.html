MORR WindowManagementModule Testpage

This webpage contains sample cases for every event generated by the MORR WindowManagementModule.
You can find a description of the expected behavior in each section.

Configuration:

  • EnabledModules must contain "MORR.Modules.WindowManagement.WindowManagementModule"
For reference, see the example configuration shipped with MORR.

Additional setup info

To make performing the tests easier, it is recommend (but not required) to close all unrelated windows before starting.

Test description

Each test consists of two items:
Action describes the action you (the tester) must perform in order to generate the relevant test data.
Expected behavior describes the event which should have resulted from your actions and thus be serialized into the recording file.
The following tests are grouped by event type.

In additional to the data described in the Expected behavior of each respective test, each event is also expected to hold this data:

  • TimeStamp: The serialized time of the event's occurence, e.g. "2020-03-16T17:43:52.517Z"
  • IssuingModule: "fab5bc0d-8b33-4dfd-9fa3-c58e0f1435b5"
If additional fields are stored but not mentioned, their data is not relevant to the test.

1. WindowFocusEvent

Test 1

Covers testcase(s): TC400

Action:
Create a Windows-Explorer window by pressing WINKEY+E. Afterwards, click anywhere on this browserwindow to focus it again.

Expected behavior:
Two new WindowFocusEvents are created with following data:

First event:

  • ProcessName: "explorer"

Second event (depending on browser):
  • Title: "MORR WindowManagementModule Testpage - Mozilla Firefox" or "MORR WindowManagementModule Testpage - Google Chrome"
  • ProcessName: "firefox" or "chrome"

2. WindowMovementEvent

Test 1

Covers testcase(s): TC410

Action:
Create a Windows-Explorer window by pressing WINKEY+E. Afterwards, grab the newly created explorer window with the left mouse button and
move it to the bottom right position of your screen(release the mouse when done).

Expected behavior:
A new WindowMovementEvent is created with following data:

  • ProcessName: "explorer"
  • OldLocation
    • X: X1
    • Y: Y1
  • NewLocation
    • X: X2
    • Y: Y2
Where X1, X2, Y1, Y2 are integer numbers and (X2, Y2) roughly being your monitor's resolution.

3. WindowResizingEvent

Test 1

Covers testcase(s): TC420

Action:
Create a Windows-Explorer window by pressing WINKEY+E and move it to the top left of your screen.
Now, grab the bottom right corner of the window with the left mousebutton and extend the window to the bottom right of your screen (release the mousebutton afterwards).

Expected behavior:
A new WindowResizingEvent is created with following data:

  • ProcessName: "explorer"
  • OldSize
    • IsEmpty: false
    • Width: X1
    • Height: Y1
  • NewSize
    • IsEmpty: false
    • Width: X2
    • Height: Y2
Where X1, X2, Y1, Y2 are integer numbers and (X2, Y2) roughly being your monitor's resolution.

4. WindowStateChangedEvent (minimize/maximize)

Test 1

Covers testcase(s): TC430

Action:
Create a Windows-Explorer window by pressing WINKEY+E. Minimize this newly created window (e.g. by clicking the minimize button in its top right corner).

Expected behavior:
A new WindowStateChangedEvent is created with following data:

  • WindowState: 1
  • ProcessName: "explorer"

Test 2

Covers testcase(s): TC430

Action:
Create a Windows-Explorer window by pressing WINKEY+E. Maximize this newly created window (e.g. by clicking the maximize button in its top right corner).

Expected behavior:
A new WindowStateChangedEvent is created with following data:

  • WindowState: 2
  • ProcessName: "explorer"
